# 02. 디자인 방향

## 한 줄 방향

**엄숙한 성서 리더가 아니라, 그룹만의 다정한 교환일기.**

이 앱은 공개 서비스가 아니고, 소규모 그룹(2~5명)이 오래 쓰는 공간이다. 따라서 첫인상은 "기능 많은 생산성 앱"보다 "다시 돌아오고 싶은 작은 방"에 가까워야 한다.

## 키워드

- 다정함
- 천천히
- 부담 없음
- 교환일기
- 기록이 쌓이는 느낌
- 파스텔이지만 유치하지 않게

## 현재 목업 기준

`together_app_mockup-1.jsx`의 방향을 기준으로 삼는다.

- 모바일 우선, 최대 너비 약 480px
- 따뜻한 크림/핑크 계열 배경
- 작은 점 패턴과 반짝이 장식
- 부드러운 화면 전환
- 둥근 카드와 말랑한 버튼
- 제목에는 귀여운 display font, 본문에는 읽기 쉬운 sans-serif

## 색상

색상은 두 층으로 나눈다.

- **작성자 인식 색:** 누가 쓴 글인지 바로 알아보게 하는 진한 색
- **분위기 색:** 배경, 장식, 선택 상태에 쓰는 연한 파스텔 색

코멘트, 하이라이트 작성자 표시는 작성자 인식 색을 우선한다. 파스텔 색은 예쁘지만 글자 대비가 약할 수 있으므로 긴 텍스트의 핵심 색으로 쓰지 않는다.

### 공통 배경

- 기본 배경: 따뜻한 크림 + 살짝 핑크
- 표면: 흰색 또는 아주 연한 파스텔
- 본문 텍스트: 어두운 자주/브라운 계열
- 보조 텍스트: 연한 자주/회색 계열

### 멤버 색상 팔레트 (`lib/color-palette.ts`)

그룹을 만들거나 참여할 때 각자 아래 6색 중 하나를 고른다. 같은 그룹 안에서는 색상이 겹칠 수 없다(서버에서 검증).

| key | accent | deep | soft |
|-----|--------|------|------|
| `olive` | `#5F6F3E` | `#48552F` | `#E8E5D4` |
| `pink` | `#A93F62` | `#8F2F50` | `#FCE4EC` |
| `indigo` | `#4C5C9E` | `#38427A` | `#E3E6F5` |
| `teal` | `#3E8E82` | `#2E6B62` | `#DCEFEA` |
| `amber` | `#B0752E` | `#8C5A20` | `#F5E9D6` |
| `plum` | `#7A4B8C` | `#5E3A6C` | `#EFE1F2` |

`olive`/`pink`는 최초 사용자였던 주환/희진의 색을 그대로 유지한 값이다.

## 작성자 색 사용 규칙

누가 쓴 글인지 색상으로 인식되게 한다.

- 글자 위에 색을 올릴 때는 대비를 위해 흰색 텍스트를 쓴다.
- 흰 카드 위에서는 작성자 이름, 왼쪽 보더, 작은 배지, 아이콘 색에 작성자 색을 쓴다.
- 긴 본문 전체를 색칠하지 않는다. 본문은 읽기 쉬운 어두운 기본 텍스트를 유지한다.
- 파스텔 색은 배경, 선택된 하이라이트, 장식, 비활성 상태에 사용한다.

## 화면 구조

로그인 후 앱은 하단 탭을 중심으로 움직인다.

- **오늘:** 지금 함께 읽는 장(또는 계획표 범위)과 그룹원의 읽음 상태
- **코멘트:** 장을 고르고 코멘트, 답글, 하이라이트를 남기는 작업 공간
- **기록:** 과거 코멘트, 하이라이트 회고

헤더에는 알림(종 아이콘)과 그룹 정보(사람 아이콘 — 초대코드 확인, 멤버 이름/그룹 이름 변경) 진입점이 있다.

`기록`은 책 목록에서 시작해 장 목록과 장별 기록 상세로 들어가는 읽기 전용 구조를 기본으로 한다. 기록 탭에 처음 들어갔을 때 기본 선택되는 책은 항상 **오늘 읽는 책**이다(하드코딩된 책으로 고정되지 않는다).

## 반응형 레이아웃

### 모바일

- 전체 앱은 한 손으로 쓰기 편한 단일 컬럼 구조
- 하단 탭으로 `오늘`, `코멘트`, `기록` 이동
- "다음 책" 카드는 오늘 탭 안에서도 보이게 한다(데스크톱 사이드바가 없는 화면 폭이라서).

### 웹/태블릿

- 넓은 화면에서는 읽기/코멘트 영역 옆에 "다음 책" 카드를 고정된 사이드바로 둔다.
- 아주 넓은 화면에서도 본문 폭은 과하게 넓히지 않는다.

## 성서 화면

성서 본문은 앱 안에 넣지 않는다. segment 화면에서 `본문 보기` 버튼을 누르면 워치타워 온라인 라이브러리의 **신세계역 성경 연구용** 해당 장을 새 창으로 연다.

성서 화면의 핵심은 본문 표시가 아니라 다음이다.

- 현재 장(또는 계획표 오늘 범위) 정보
- 그룹원의 읽음 체크 상태
- 하이라이트 기록
- 장별 코멘트
- 답글과 작은 반응

장별 코멘트는 빠르게 흘러가면 안 된다. 한 달 뒤 다시 봐도 맥락이 남아 있는 기록처럼 보여야 한다.

## 구절 표시 UI

성경 전체 1,189장의 절 수 데이터가 채워져 있으므로, 어떤 책/장을 펼쳐도 절 번호 버튼 그리드로 구절을 선택한다(시작 절 클릭 → 끝 절 클릭으로 범위 선택). 데이터가 없는 예외적인 경우에만 직접 타이핑 입력으로 대체된다.

## 그룹 정보 화면

헤더의 사람 아이콘으로 여는 drawer다.

- 그룹 이름 (그룹장만 연필 아이콘으로 수정 가능)
- 초대코드 (큰 글씨 + 복사 버튼) — 그룹원 누구나 언제든 다시 확인 가능
- 멤버 목록 (색상 점 + 이름, 본인 행에만 연필 아이콘으로 이름 수정 가능)

## 문구 톤

피해야 할 말:

- "진도가 밀렸어요"
- "오늘 목표를 달성하세요"
- "X일째 읽지 않았어요"
- "상대보다 늦어요"
- 지나치게 다정하거나 오글거리는 문구("여유 있을 때 천천히, 함께." 류) — 실사용 피드백으로 제외됨. 특별히 할 말(기다리는 중, 모두 읽음)이 없으면 문구 자체를 비워둔다.

좋은 톤:

- "오늘은 쉬어가도 괜찮아요"
- "함께 읽는 중"
- "기다리는 중"
- "지금까지 함께 읽은 47장"

## UI 원칙

- 모바일에서 먼저 예쁘고 편해야 한다.
- 카드 안에 카드를 과하게 중첩하지 않는다.
- 큰 수치 경쟁이나 전체 퍼센트는 보여주지 않는다.
- 색은 귀엽게 쓰되, 화면 전체가 한 색으로만 보이지 않게 한다.
- 버튼과 입력창은 손가락으로 누르기 편해야 한다.
- 애니메이션은 부드럽게, 하지만 기능을 방해하지 않게 쓴다.
- 서버 호출 중이라고 해서 전체 화면을 막거나 마우스 커서를 로딩 상태로 바꾸지 않는다.
- 읽음 체크, 반응, 다음 책 지정, 코멘트/하이라이트 작성, 선물 저장은 화면 반응을 먼저 보여주고 저장은 뒤따라가게 한다.
- 로그인처럼 사용자가 정말 기다려야 하는 동작은 버튼 안 문구로만 작게 상태를 보여준다.
