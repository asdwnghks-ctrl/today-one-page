# 02. 디자인 방향

## 한 줄 방향

**엄숙한 성서 리더가 아니라, 둘만의 다정한 교환일기.**

이 앱은 공개 서비스가 아니고, 딱 두 사람이 오래 쓰는 공간이다. 따라서 첫인상은 "기능 많은 생산성 앱"보다 "다시 돌아오고 싶은 작은 방"에 가까워야 한다.

## 키워드

- 다정함
- 천천히
- 부담 없음
- 교환일기
- 기록이 쌓이는 느낌
- 파스텔이지만 유치하지 않게
- 성서 읽기와 일상 대화가 함께 있는 공간

## 현재 목업 기준

`together_app_mockup-1.jsx`의 방향을 기준으로 삼는다.

- 모바일 우선, 최대 너비 약 480px
- 따뜻한 크림/핑크 계열 배경
- 작은 점 패턴과 반짝이 장식
- 부드러운 화면 전환
- 둥근 카드와 말랑한 버튼
- 제목에는 귀여운 display font, 본문에는 읽기 쉬운 sans-serif

## 색상

색상은 두 층으로 나눈다.

- **작성자 인식 색:** 누가 쓴 글인지 바로 알아보게 하는 진한 색
- **분위기 색:** 배경, 장식, 선택 상태에 쓰는 연한 파스텔 색

채팅, 코멘트, 하이라이트 작성자 표시는 작성자 인식 색을 우선한다. 파스텔 색은 예쁘지만 글자 대비가 약할 수 있으므로 긴 텍스트의 핵심 색으로 쓰지 않는다.

### 공통 배경

- 기본 배경: 따뜻한 크림 + 살짝 핑크
- 표면: 흰색 또는 아주 연한 파스텔
- 본문 텍스트: 어두운 자주/브라운 계열
- 보조 텍스트: 연한 자주/회색 계열

### 희진

- 작성자 색: 어두운 핑크 `#A93F62`
- 작성자 색 hover/active: 깊은 핑크 `#8F2F50`
- 분위기 색: 파스텔 핑크 `#F4B5C9`
- 보조: 라벤더 `#C8B5E8`
- 보조: 하늘 `#B5D5E8`
- 연한 배경: `#FCE4EC`

### 주환

- 작성자 색: 어두운 올리브 `#5F6F3E`
- 작성자 색 hover/active: 깊은 올리브 `#48552F`
- 분위기 색: 올리브 `#7A8456`
- 보조: 네이비 `#3D4A5C`
- 보조: 브라운 `#8B6F47`
- 연한 배경: `#E8E5D4`

## 작성자 색 사용 규칙

누가 쓴 글인지 색상으로 인식되게 한다.

- 희진이 쓴 글: 어두운 핑크 `#A93F62`
- 주환이 쓴 글: 어두운 올리브 `#5F6F3E`
- 글자 위에 색을 올릴 때는 대비를 위해 흰색 텍스트를 쓴다.
- 흰 카드 위에서는 작성자 이름, 왼쪽 보더, 작은 배지, 아이콘 색에 작성자 색을 쓴다.
- 긴 본문 전체를 색칠하지 않는다. 본문은 읽기 쉬운 어두운 기본 텍스트를 유지한다.
- 파스텔 색은 배경, 선택된 하이라이트, 장식, 비활성 상태에 사용한다.

## 화면 구조

로그인 후 앱은 하단 탭을 중심으로 움직인다.

- **오늘:** 지금 함께 읽는 장과 둘의 읽음 상태
- **읽기:** 책 선택, 장 목록, 과거 장 다시 보기
- **채팅:** 둘만의 실시간 대화
- **기록:** 과거 코멘트, 하이라이트, 메시지 회고

MVP에서는 `오늘`, `읽기`, `채팅`을 먼저 만든다. `기록`은 처음엔 간단한 목록으로 시작하거나 다음 단계로 미룬다.

## 성서 화면

성서 본문은 앱 안에 넣지 않는다. segment 화면에서 `본문 보기` 버튼을 누르면 jw.org의 해당 장을 새 창으로 연다.

성서 화면의 핵심은 본문 표시가 아니라 다음이다.

- 현재 장 정보
- 둘의 읽음 체크 상태
- 하이라이트 기록
- 장별 코멘트
- 답글과 작은 반응

장별 코멘트는 채팅처럼 빠르게 흘러가면 안 된다. 한 달 뒤 다시 봐도 맥락이 남아 있는 기록처럼 보여야 한다.

## 채팅 화면

채팅은 카톡 대용에 가깝지만, 압박을 줄이는 방향으로 만든다.

- 입력창은 하단 고정
- 내 메시지와 상대 메시지는 말풍선으로 구분
- 내 말풍선은 현재 로그인한 사람의 작성자 색을 사용하고 텍스트는 흰색으로 둔다.
- 상대 말풍선은 흰색 또는 아주 연한 배경을 사용하고, 이름/테두리/작은 표시에는 상대의 작성자 색을 쓴다.
- 날짜 구분선은 작고 조용하게
- 읽음 표시는 작고 부드럽게
- 온라인/오프라인, 마지막 접속 시간, 타이핑 중 표시는 넣지 않는다
- 새 메시지는 실시간으로 나타난다
- 메시지는 영구 보관한다

채팅은 성서 코멘트와 섞지 않는다. 채팅은 일상의 흐름이고, 성서 코멘트는 특정 장에 묶인 깊은 기록이다.

## UI 원칙

- 모바일에서 먼저 예쁘고 편해야 한다.
- 카드 안에 카드를 과하게 중첩하지 않는다.
- 큰 수치 경쟁이나 전체 퍼센트는 보여주지 않는다.
- 색은 귀엽게 쓰되, 화면 전체가 한 색으로만 보이지 않게 한다.
- 버튼과 입력창은 손가락으로 누르기 편해야 한다.
- 애니메이션은 부드럽게, 하지만 기능을 방해하지 않게 쓴다.
