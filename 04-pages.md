# 04. 페이지 흐름

## 한 줄 방향

**처음 들어와서, 오늘 읽고, 기록을 남기고, 자연스럽게 대화하는 흐름.**

이 앱은 복잡한 메뉴보다 반복 사용의 편안함이 중요하다. 화면은 적게, 상태는 분명하게, 문구는 다정하게 둔다.

## 전체 구조

로그인 전:

- 랜딩
- 사람 선택
- 비밀번호 입력

로그인 후:

- 오늘
- 읽기
- 채팅
- 기록

MVP에서는 `오늘`, `읽기`, `채팅`, `기록`을 모두 구현한다. `기록`은 성경 책 목록에서 시작하는 탐색 화면으로 만든다.

## 공통 내비게이션

모바일에서는 하단 탭을 쓴다.

- 오늘
- 읽기
- 채팅
- 기록

웹/태블릿에서는 화면 폭에 따라 달라진다.

- 좁은 화면: 모바일과 같은 하단 탭
- 중간 화면: 하단 탭 또는 왼쪽 얇은 내비게이션
- 넓은 화면: 읽기/기록 영역을 가운데에 두고, 채팅은 버튼으로 여닫는 오른쪽 drawer 또는 floating panel로 연다.

하단 탭은 기능 설명을 길게 쓰지 않고, 짧은 이름과 아이콘으로 충분하게 만든다.

## 1. 랜딩

목적:

- 앱의 첫인상을 만든다.
- 두 사람 중 누가 들어가는지 고르게 한다.

구성:

- 앱 이름: 오늘도 한 페이지
- 짧은 문구: 누구로 들어갈까요?
- 주환 카드
- 희진 카드

동작:

- 사람 카드를 누르면 로그인 화면으로 이동한다.
- 회원가입 버튼은 없다.
- 공개 소개나 마케팅 문구는 넣지 않는다.

상태:

- 선택 전
- 선택 후 로그인으로 이동

## 2. 로그인

목적:

- 선택한 사람이 본인인지 확인한다.

구성:

- 선택한 사람 이름
- 비밀번호 입력
- 들어가기 버튼
- 뒤로가기

동작:

- 비밀번호가 맞으면 앱으로 들어간다.
- 틀리면 부드러운 오류 문구를 보여준다.
- 오류 문구는 위협적이지 않게 쓴다.

문구 예:

- "비밀번호를 다시 확인해 주세요"
- "조금만 다시 입력해 볼까요?"

구현 방식:

- Supabase Auth를 내부적으로 사용한다.
- 화면은 이름 선택 + 비밀번호 입력처럼 보이게 한다.

## 3. 오늘

목적:

- 지금 둘이 함께 읽고 있는 segment를 보여준다.
- 한 번의 탭으로 본문을 열고, 읽음 체크를 할 수 있게 한다.

구성:

- 오늘 날짜
- 현재 사용자 표시
- 현재 책/장
- 분류 이름
- 본문 보기 버튼
- 읽음 체크 버튼
- 둘의 읽음 상태
- 최근 코멘트/하이라이트 미리보기
- 채팅으로 가는 작은 진입점

상태:

- 아무도 읽지 않음
- 내가 읽음, 상대 기다리는 중
- 상대가 읽음, 내가 아직 안 읽음
- 둘 다 읽음, 다음 장으로 넘어갈 수 있음
- 책을 다 읽음, 다음 책 고르기 필요
- 다음 책 제안 대기 중
- 다음 책 수락 대기 중

문구 예:

- "여유 있을 때 천천히"
- "함께 읽는 중"
- "희진을 기다리는 중"
- "주환을 기다리는 중"
- "둘 다 읽었어요"
- "다음 장으로 가볼까요?"

하지 않을 것:

- "진도가 밀렸어요"
- "오늘 목표 실패"
- "X일째 미완료"
- 전체 성서 대비 퍼센트

## 4. 읽기

목적:

- 책을 고르고, 현재 책의 장 목록을 보고, 과거 segment로 돌아간다.

구성:

- 현재 읽는 책
- 책 선택
- section별 책 목록
- 책 안의 segment 목록
- 각 segment의 상태
- 본문 보기
- segment 상세 진입

segment 상태:

- 아직 열리지 않음
- 읽는 중
- 내가 읽음
- 상대가 읽음
- 둘 다 읽음
- 코멘트 있음
- 하이라이트 있음

책 선택 흐름:

1. 현재 책을 완료한다.
2. 다음 책 선택 화면을 연다.
3. 한 사람이 다음 책을 제안한다.
4. 상대에게 제안 카드가 보인다.
5. 상대가 수락하면 새 책을 시작한다.

시작 책은 **전도서**다. 이후부터는 제안/수락 흐름을 사용한다.

## 5. Segment 상세

목적:

- 특정 장의 기록을 남기고 다시 읽는다.

구성:

- section 이름
- 책/장 제목
- 둘의 읽음 상태
- 본문 보기 링크
- 구절 표시 버튼
- 코멘트 작성 버튼
- 하이라이트 목록
- 코멘트 목록
- 답글
- 반응

동작:

- 본문 보기는 WOL의 신세계역 성경 연구용 해당 장을 새 창으로 연다.
- 구절 표시는 구절 참조, 색, 짧은 메모를 저장한다.
- 코멘트는 긴 글도 쓸 수 있게 한다.
- 답글은 highlight와 comment 모두에 달 수 있다.
- 반응은 작고 가볍게 붙인다.

정렬:

- 기본은 오래된 순 또는 자연스러운 대화 순으로 보여준다.
- 최근 작성한 항목이 아래에 붙는 방식이 교환일기 느낌에 잘 맞는다.
- 기록 화면에서는 최신순으로 다시 볼 수 있다.

## 6. 채팅

목적:

- 카톡 대용에 가까운 둘만의 실시간 대화.

모바일 구성:

- 채팅 헤더
- 상대 접속 상태
- 날짜 구분선
- 메시지 목록
- 읽음 표시
- 입력창

웹 구성:

- 넓은 화면에서도 버튼으로 여닫는 오른쪽 drawer 또는 floating panel
- 읽기 화면과 동시에 볼 수 있음
- 중간 화면에서는 drawer 또는 floating panel

동작:

- 새 메시지는 실시간으로 반영된다.
- 내가 보낸 마지막 메시지에만 작게 읽음 표시를 둔다.
- 상대가 접속 중이면 작게 표시한다.
- 타이핑 중 표시는 넣지 않는다.
- 마지막 접속 시간은 넣지 않는다.

메시지 보존:

- 모든 메시지는 영구 보관한다.
- 본인 메시지는 수정/삭제할 수 있다.
- 삭제는 가능하면 soft delete로 처리한다.

## 7. 기록

목적:

- 시간이 지난 뒤 다시 꺼내보는 공간.

MVP 구성:

- 성경 책 목록
- 책별 읽음 요약
- 책을 눌렀을 때 장 목록
- 각 장 옆의 읽음 표시
- 장을 눌렀을 때 장별 기록 상세
- 그 장을 언제 읽었는지
- 그 장에 달았던 코멘트
- 그 장에 달았던 하이라이트
- 코멘트/하이라이트의 답글과 반응

MVP 다음 단계:

- 사람별 필터
- 반응이 붙은 글
- 날짜별 회고
- "작년 오늘 우리는"
- 채팅 검색

기록 화면은 앱의 장기 가치를 만드는 부분이다. 처음부터 화려할 필요는 없지만, 데이터가 쌓이면 반드시 좋아져야 한다.

## 8. 설정

MVP에서는 별도 탭으로 두지 않는다.

필요한 설정은 작은 메뉴 안에 넣는다.

- 로그아웃
- 테마 확인
- 비밀번호 변경
- 앱 정보

푸시 알림, 이메일 알림, 계정 삭제 같은 기능은 MVP 이후 필요가 생기면 추가한다.

## 주요 사용자 흐름

### 처음 들어오는 흐름

1. 랜딩에서 본인 이름을 누른다.
2. 비밀번호를 입력한다.
3. 오늘 화면으로 들어온다.
4. 현재 장을 확인한다.
5. 본문 보기를 눌러 WOL 신세계역 성경 연구용 페이지에서 읽는다.
6. 돌아와서 "다 읽었어요"를 누른다.
7. 코멘트나 하이라이트를 남긴다.

### 먼저 읽은 사람의 흐름

1. 오늘 장을 읽는다.
2. 읽음 체크를 한다.
3. "상대를 기다리는 중" 상태를 본다.
4. 기다리는 동안 코멘트나 하이라이트를 남긴다.
5. 채팅으로 일상 대화를 이어간다.

### 나중에 읽은 사람의 흐름

1. 오늘 화면에서 상대가 먼저 읽은 것을 본다.
2. 본문을 읽는다.
3. 상대가 남긴 기록을 본다.
4. 답글이나 반응을 남긴다.
5. 읽음 체크를 한다.
6. 다음 장이 열린다.

### 과거 기록을 보는 흐름

1. 기록 탭에 들어간다.
2. 성경 책 목록에서 책을 고른다.
3. 장 목록에서 읽음 표시를 확인한다.
4. 장을 누른다.
5. 그 장을 언제 읽었는지 확인한다.
6. 그때 남긴 코멘트, 하이라이트, 답글을 본다.
7. 새 코멘트를 덧붙일 수 있다.

## 빈 상태 문구

코멘트 없음:

- "아직 남긴 생각이 없어요"
- "읽다가 떠오르는 게 있으면 천천히 남겨요"

하이라이트 없음:

- "표시한 구절이 아직 없어요"
- "마음에 남는 구절을 살짝 표시해 둘 수 있어요"

채팅 없음:

- "둘만의 대화를 시작해 볼까요?"

기록 없음:

- "기록은 천천히 쌓이면 돼요"

## 결정된 것

- 기록 탭은 MVP에 포함한다.
- 다음 책은 한 사람이 제안하고 다른 사람이 수락한다.
- 웹 채팅은 버튼으로 여닫는 drawer 또는 floating panel로 둔다.
