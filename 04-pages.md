# 04. 페이지 흐름

## 한 줄 방향

**그룹으로 들어와서, 오늘 읽고, 기록을 남기는 흐름.**

이 앱은 복잡한 메뉴보다 반복 사용의 편안함이 중요하다. 화면은 적게, 상태는 분명하게, 문구는 다정하게 둔다.

## 전체 구조

로그인 전 (`authMode` 상태로 분기, `app/page.tsx` 한 컴포넌트 안에서 처리):

- 랜딩 (그룹 만들기 / 초대코드로 들어가기)
- 그룹 생성 (그룹명, 내 이름/색상, 읽기 방식, 시작 지점, PIN)
- 초대코드 입력 → 멤버 선택
- 새 멤버로 참여 (이름/색상, PIN)
- 그룹 생성 직후 초대코드 확인 화면

로그인 후:

- 오늘
- 코멘트
- 기록

`채팅` 탭은 없다. 원래 MVP 설계에 있었지만 완전히 제거했다.

## 공통 내비게이션

모바일에서는 하단 탭을 쓴다.

- 오늘
- 코멘트
- 기록

웹/태블릿에서는 화면 폭에 따라 달라진다.

- 좁은 화면: 모바일과 같은 하단 탭
- 넓은 화면: 읽기/기록 영역 옆에 "다음 책" 카드를 사이드바로 둔다(하루 1장 모드만 해당, 계획표 모드는 사이드바 없음).

헤더에는 항상 알림(종) 아이콘과 그룹 정보(사람) 아이콘이 있다.

## 1. 랜딩

목적:

- 새로 온 사람이 그룹을 만들지, 이미 있는 그룹에 들어갈지 정하게 한다.

구성:

- 앱 이름: 오늘도 한 페이지
- "그룹 만들기" 버튼
- "초대코드로 들어가기" 버튼

동작:

- 각 버튼을 누르면 해당 흐름으로 이동한다.
- 회원가입/공개 소개 문구는 없다.

## 2. 그룹 생성

목적:

- 새 그룹을 만들고 초대코드를 발급받는다.

구성:

- 그룹 이름 입력 (placeholder: "예: 주니네 가족")
- 내 이름 입력
- 색상 선택 (6색 팔레트)
- 읽기 방식 선택: 하루 1장 / 읽기 계획표
- **시작 지점 선택** (읽기 방식에 따라 달라짐):
  - 하루 1장: 시작할 책 드롭다운 (`GET /api/books`로 66권 목록을 미리 받아온다 — 로그인 전이라 인증이 필요 없는 공개 엔드포인트)
  - 읽기 계획표: 11개 범주 드롭다운 (`lib/reading-plan-sections.ts`)
- 내 비밀번호(PIN, 4~8자) 입력
- "그룹 만들기" 버튼

동작:

- `POST /api/groups`로 제출한다. 성공하면 바로 로그인 상태로 넘어가지 않고, **초대코드 확인 화면**을 먼저 보여준다.
- 이 화면에서 곧바로 앱 상태를 새로고침(`fetchState`)하지 않는다 — 예전에는 생성 직후 바로 새로고침을 해버려서 초대코드가 뜨자마자 사라지는 버그가 있었다(실사용 피드백으로 발견/수정).

## 2-1. 초대코드 확인 (그룹 생성 직후)

목적:

- 방금 발급된 초대코드를 사용자가 실제로 읽고 복사할 시간을 준다.

구성:

- 큰 글씨의 초대코드
- 복사 버튼
- "시작하기" 버튼

동작:

- "시작하기"를 눌러야 비로소 `fetchState()`가 호출되어 로그인 상태(오늘 화면)로 넘어간다.
- 이후에도 초대코드는 그룹 정보 화면(§9)에서 언제든 다시 확인할 수 있다.

## 3. 초대코드 입력 → 멤버 선택

목적:

- 이미 있는 그룹에 로그인하거나, 새 멤버로 참여한다.

구성:

- 초대코드 입력
- (조회 성공 시) 그룹 이름 + 멤버 이름 카드 목록
- "새 멤버로 참여하기" 버튼 (인원이 다 안 찼을 때만 노출)
- 선택한 멤버의 비밀번호 입력

동작:

- `GET /api/groups/lookup?code=...`로 그룹명/멤버 목록을 조회한다(PIN 해시는 응답에 포함 안 됨).
- 멤버 카드를 누르면 비밀번호 입력란이 뜨고, 맞으면 `POST /api/login`으로 로그인한다.
- 코드가 틀리면 "초대 코드를 다시 확인해 주세요" 같은 부드러운 오류를 보여준다.

## 4. 새 멤버로 참여

목적:

- 초대코드를 가진 사람이 자기 프로필을 만든다.

구성:

- 내 이름 입력
- 색상 선택 (이미 그룹에서 쓰이는 색은 비활성화되어 고를 수 없음)
- 내 비밀번호(PIN) 입력

동작:

- `POST /api/join`으로 제출한다. 인원이 다 찼거나 색상이 중복되면 에러를 보여준다.
- 성공하면 로그인 상태로 넘어간다.

## 5. 오늘

목적:

- 지금 그룹이 함께 읽고 있는 범위를 보여준다.
- 한 번의 탭으로 본문을 열고, 읽음 체크를 할 수 있게 한다.

구성:

- 현재 사용자 표시
- 현재 책/장 (하루 1장 모드) 또는 오늘의 범위(계획표 모드, 여러 장/두 책에 걸칠 수 있음 — 예: "오바댜 1장 · 요나 1~4장")
- 본문 보기 버튼
- 읽음 체크 버튼 (계획표 모드는 그 날의 모든 segment를 한 번에 체크)
- 그룹원별 읽음 상태 (사람 수만큼 카드가 늘어남)
- 현재 회차의 못 읽음 횟수
- 현재 회차의 선물 약속 상태
- 최근 코멘트/하이라이트 미리보기
- "다음 책" 카드 (하루 1장 모드만, 모바일에서는 오늘 탭 안에도 표시됨)

상태:

- 아무도 읽지 않음
- 일부만 읽음, 나머지 기다리는 중
- 전원 읽음, 다음 한국시간 오전 2시 경계 이후 다음으로 넘어감
- 누군가 전날 못 읽어서 멈춰 있음 → 읽은 사람이 "다음 범위 열기"(수동 진행)를 쓸 수 있음
- 계획표 완독(363일차 완료) → "완독했어요" 상태

문구 예:

- "함께 읽는 중"
- "오늘은 모두 읽었어요. 새벽 2시에 다음 범위가 열려요."
- "OOO님이 전날 못 읽어서 멈춰 있어요."

특별히 할 말이 없으면(기다리는 중도 아니고 전원 읽음도 아닌 평범한 상태) **문구 자체를 띄우지 않는다.** 예전에는 "여유 있을 때 천천히, 함께." 같은 채움 문구가 있었지만 실사용 피드백("느끼하다")으로 제거했다.

하지 않을 것:

- "진도가 밀렸어요"
- "오늘 목표 실패"
- "X일째 미완료"
- 전체 성서 대비 퍼센트
- 서버 호출 중이라는 이유로 전체 화면을 막거나 마우스 커서를 로딩으로 바꾸는 것

## 6. 코멘트

목적:

- 책과 장을 고르고, 해당 장에 코멘트와 답글을 남긴다.

구성:

- 현재 읽는 책
- 책 선택 (section별)
- 책 안의 segment 목록
- 각 segment의 상태
- 본문 보기
- segment 상세 진입
- 코멘트 작성
- 코멘트 답글 작성
- (하루 1장 모드) "다음 책" 카드도 여기 사이드바에 노출

## 7. Segment 상세

목적:

- 특정 장의 기록을 남기고 다시 읽는다.

구성:

- section 이름
- 책/장 제목
- 그룹원별 읽음 상태
- 본문 보기 링크
- 구절 표시 버튼 — **전체 책/장에서 절 번호 버튼 그리드로 선택 가능** (예전엔 전도서만 버튼이고 나머지는 직접 타이핑이었는데, 전체 책 절 수 데이터를 채워서 통일함)
- 코멘트 작성 버튼
- 하이라이트 목록
- 코멘트 목록
- 답글
- 반응

동작:

- 본문 보기는 WOL의 신세계역 성경 연구용 해당 장을 새 창으로 연다.
- 구절 표시는 구절 참조, 색, 짧은 메모를 저장한다.
- 답글은 highlight와 comment 모두에 달 수 있다.

## 8. 기록

목적:

- 시간이 지난 뒤 다시 꺼내보는 공간.

구성:

- 성경 책 목록
- 책별 읽음 요약
- 책을 눌렀을 때 장 목록
- 각 장 옆의 읽음 표시
- 장을 눌렀을 때 장별 기록 상세
- 그 장을 언제 읽었는지
- 그 장에 달았던 코멘트/하이라이트
- 답글과 반응

기본 선택되는 책은 **오늘 읽는 책**이다. 사용자가 직접 다른 책을 누르면 그 선택을 유지한다(자동으로 되돌리지 않음). 예전에는 항상 전도서로 고정되어 있었다(2인 전용 시절, 시작 책이 전도서였던 흔적).

기록 탭에서는 새 코멘트 작성, 수정, 삭제를 하지 않는다.

## 9. 그룹 정보

목적:

- 초대코드를 언제든 다시 확인하고, 이름을 관리한다.

구성:

- 그룹 이름 (그룹장이면 연필 아이콘으로 인라인 수정 가능, 아니면 텍스트만)
- 초대코드 (큰 글씨 + 복사 버튼)
- "코드를 잃어버려도 로그인된 멤버가 있으면 이 화면에서 다시 확인할 수 있어요" 안내
- 멤버 목록 (색상 점 + 이름, 본인 행에만 연필 아이콘으로 이름 수정 가능)

동작:

- 헤더의 사람 아이콘을 누르면 drawer로 열린다.
- 그룹 이름 변경은 그룹장만(`update_group_name`), 본인 이름 변경은 누구나 자기 것만(`update_my_name`).

## 10. 다음 책 (하루 1장 모드 전용)

목적:

- 현재 책이 끝난 뒤 무엇을 읽을지 보여주고, 그룹장이 바꿀 수 있게 한다.

구성:

- "다음 책: OOO" (자동으로 정해졌는지, 그룹장이 정했는지 문구로 구분)
- 그룹장에게만: 드롭다운 + "다음 책 바꾸기" 버튼

동작:

- 지정이 없으면 서버가 자동으로 정한다(안 읽은 책 중 정렬 순서 우선, 전부 읽었으면 창세기부터 다시).
- 그룹장이 지정하면 그 책이 우선한다. 그룹이 막 만들어졌거나 레거시로 멈춰 있던 상태라면 그 자리에서 즉시 시작된다.
- 예전에는 "한 사람이 제안 → 다른 사람이 수락" 하는 2인 전용 흐름이었지만, N명 그룹에 안 맞아서 제거했다.

## 11. 알림

목적:

- 새 코멘트, 답글, 읽음 체크, 다음 책 지정을 조용히 알려준다.

구성:

- 헤더의 작은 unread badge
- 알림 목록
- 관련 화면으로 이동하는 항목
- 모두 읽음 처리 버튼

동작:

- 목록에는 읽지 않은 알림만 보여준다.
- 알림을 눌러 관련 화면으로 이동하면 해당 알림을 읽음 처리한다.
- (제거됨) 채팅 관련 알림/이동은 더 이상 없다.

## 12. 설정

별도 탭으로 두지 않는다.

- 로그아웃 (헤더)
- 이름/그룹 이름 변경은 그룹 정보 화면에서 (§9)

## 주요 사용자 흐름

### 처음 그룹을 만드는 흐름

1. 랜딩에서 "그룹 만들기"를 누른다.
2. 그룹명, 내 이름/색상, 읽기 방식, 시작 지점, 비밀번호를 정한다.
3. 초대코드를 확인하고 복사한다.
4. "시작하기"를 누르면 바로 오늘 화면에 읽을거리가 떠 있다(빈 상태 없음).

### 초대코드로 합류하는 흐름

1. 랜딩에서 "초대코드로 들어가기"를 누른다.
2. 코드를 입력한다.
3. 그룹 멤버 목록에서 본인이 없으면 "새 멤버로 참여하기"를 누른다.
4. 이름/색상/비밀번호를 정하고 참여한다.

### 다시 로그인하는 흐름

1. 초대코드를 입력한다.
2. 멤버 목록에서 본인 이름을 누른다.
3. 비밀번호를 입력한다.

### 다음 책이 자동으로 넘어가는 흐름

1. 현재 책의 마지막 장을 전원이 읽는다.
2. 다음 한국시간 오전 2시 경계가 지난다.
3. 선물이 공개된다(현재 회차 것만).
4. 그룹장이 미리 지정해둔 책 또는 자동으로 정해진 책의 1장이 열린다.
5. 아무도 개입하지 않아도 자연스럽게 이어진다.

### 과거 기록을 보는 흐름

1. 기록 탭에 들어간다 (오늘 읽는 책이 기본 선택되어 있음).
2. 다른 책을 보고 싶으면 책 목록에서 고른다.
3. 장 목록에서 읽음 표시를 확인한다.
4. 장을 누른다.
5. 그때 남긴 코멘트, 하이라이트, 답글을 본다.
6. 새 코멘트를 덧붙이고 싶으면 `코멘트` 탭으로 이동한다.

## 빈 상태 문구

코멘트 없음:

- "아직 남긴 생각이 없어요"

하이라이트 없음:

- "표시한 구절이 아직 없어요"

기록 없음:

- "기록은 천천히 쌓이면 돼요"

## 결정된 것

- 기록 탭은 오늘 읽는 책을 기본 선택으로 연다.
- 다음 책은 그룹장이 지정하거나 자동으로 정해진다(제안/수락 협상 없음).
- 채팅 탭/화면은 없다.
- 그룹 정보 화면에서 초대코드 확인, 이름 변경(본인), 그룹 이름 변경(그룹장)을 처리한다.
